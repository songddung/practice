# MST(최소 신장 트리)
    - 양방향 그래프에서 최소한의 간선으로
        모든 노드를 연결하는 트리
        ( 노드의 수 N개일때 N-1의 간선 )

    - Prim: 정점을 기준으로 작은 가중치부터 고르자
        - 우선순위큐 사용
        - 시간복잡도: ( VlogV + ElogE )
            - VlogV
            - ElogV
            -> 힙 트리의 높이가 최적화를 하지 않아도
                logE 만큼 높아지지 않는다.
            -> 그 만큼 높아지지 않는다
            - > 그 만큼 시간이 빨라진다.
        - 결론 : 간선이 많을 수록 유리하다.
        

    - Kruskal : 간선을 기준으로 작은 가중치부터 고르자
        - 정렬 사용
        - 간선이 E개 라면 시간은 O(ElogE)
            + 사이클 검사(유니온파인드)
            -> 거의 상수시간
        - 결론 : 정점에 간선이 적을수록 유리하다
